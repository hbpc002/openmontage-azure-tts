## Lessons Learned / Known Pitfalls

### Remotion + Vertical Video
- **Entry point:** Must use the file that calls `registerRoot()` (e.g. `index.tsx`), NOT `Root.tsx`, as the CLI entry point. Otherwise bundler errors.
- **OffthreadVideo trimming:** `trimBeforeSeconds`/`trimAfterSeconds` in scene props are NOT automatically passed to `<OffthreadVideo startFrom={...}>`. The SceneVideo component must explicitly wire these up — currently it doesn't.
- **Caption sync:** Word-level captions generated by proportional timing (without real ASR) drift from audio. Use sentence-level captions instead (`SentenceCaption` component) for reliable sync.

### Pexels Video Downloads
- **HD vs SD:** Default `preferred_quality` is `"sd"` which returns 360-540p videos. For 1080×1920 output, explicitly set `size: "large"` + `orientation: "portrait"` to get 2K-4K sources.
- **Download timeouts:** 15+ videos downloaded serially will timeout. Batch in groups of 5 or use parallel downloads.
- **Video-content matching:** Always map scenes to narration sentences and verify no repeats. Pexels search often returns imperfect matches — verify before rendering.

### General
- **Opening title overlays:** A title scene at 0s overlapping the first video scene makes the user think the video has no content. Move titles to the end or keep them very brief (<1.5s) and non-blocking.
- **Scene reuse:** The same video file used in multiple scenes is immediately noticeable. Always use unique clips per scene.
- **Font size for 1920×1080 video:** Default CSS font sizes that look fine on a monitor are unreadable on video. Start at title-xl: 120px, body-md: 36px for legible output.
- **Emoji font installation:** Without Noto Color Emoji installed, emoji renders as tofu boxes. Install: `apt-get install -y fonts-noto-color-emoji` and add to CSS `font-family` stack BEFORE sans-serif fallbacks.
- **Re-render after edits:** The render output has a timestamp. If you edit `index.html` or audio assets after rendering, the render is stale. Always re-render after any source change.

### Azure TTS + HyperFrames Timing
- **`[停顿:X秒]` SSML markers NOT supported** by `tts-express`. To add pauses between sentences, generate separate TTS files per sentence, then use FFmpeg `concat` with silence (anullsrc) gaps between them.
- **Subtitle-to-audio sync:** Generate sentences individually at slower rate (e.g. `-10%`), then concat with calculated silence gaps so each sentence start aligns with its subtitle `data-start` in HyperFrames. Account for the overall `adelay` offset in FFmpeg mix.
- **Continuous TTS drift:** A single flat TTS of the full poem reads too fast — later sentences drift from their subtitle windows. Paced per-sentence generation is required.
- **Total duration math:** Sum sentence durations + silence gaps + FFmpeg delay. Ensure total ≤ video duration, especially for the last sentence.
- **SSML emotion wrapper order:** `<mstts:express-as>` MUST wrap `<prosody>`, not be inside it. Wrong: `<prosody><mstts:express-as>text</mstts:express-as></prosody>`. Correct: `<mstts:express-as><prosody>text</prosody></mstts:express-as>`. tts-express had the reverse order — patch the source if needed.
- **Chinese phoneme markers for polyphones:** Add `[音:pinyin]字` inline markers in tts-express to fix polyphonic character pronunciation. E.g., `干[音:gàn]活` for 干活, `银[音:xíng]` for 银行, `效[音:lǜ]` for 效率, `[音:chóng]新` for 重新. Requires patching tts-express to handle the `[音:]` syntax in SSML.
- **Azure TTS key scope:** `AZURE_SPEECH_KEY` is loaded by `tool_registry._load_dotenv()` — NOT available in shell env. Use the code tool (not shell commands) when invoking Azure TTS.

### Audio Mixing (FFmpeg)
- **`audio_mixer` truncation:** Both `mix` and `full_mix` operations truncate output to the shortest input (usually speech). Music never extends past speech. **Use FFmpeg directly** for full control.
- **Recommended FFmpeg mix pattern:**
  - `atrim=0:<duration>` to set music length
  - `adelay=<ms>|<ms>` to offset speech start
  - `amix=inputs=2:duration=first:dropout_transition=2` to layer
  - `afade=t=in/out` for smooth music intro/outro
  - `-t <video_duration>` to match composition length
- **Music-intro silence:** Add 2s of music-only before speech starts for natural pacing (adelay narration by 2000ms).

### HyperFrames Composition Duration
- **Extend composition if last sentence spills:** If the final sentence runs past the video end, increase `data-duration` on `#root` and update the last video's `data-duration` + its GSAP `vOut` fade time. A frame hold on the last scene for 1-2s is fine.
- **`data-duration` must match GSAP timeline:** The root `data-duration` attribute must be ≥ the last GSAP animation completion time. If shorter, the video truncates before narration finishes.
- **Scene `data-start` + `data-duration` continuity:** Each scene's `data-start` should equal the previous scene's `data-start + data-duration`. Gaps create black frames; overlaps cause flicker.
- **BGM must match composition length:** Loop or trim background music to exactly match `data-duration` on `#root`. Any shorter and music cuts abruptly; any longer and it's wasted.
- **Scene total must equal composition duration:** Sum of last scene's `data-start + data-duration` should approximately equal the root `data-duration`. Recalculate when adjusting any single scene's timing.
